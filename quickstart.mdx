## Prerequisites

Before you begin, ensure you have the following installed on your system:

- **Node.js**: Version 18.0 or higher ([Download Node.js](https://nodejs.org/))
- **npm**: Version 9.0 or higher (comes with Node.js)
- **Git**: For cloning the repository
- **Code Editor**: VS Code recommended

<Info>
  You can verify your Node.js and npm versions by running `node --version` and `npm --version` in your terminal.
</Info>

## Installation

<Steps>
  <Step title="Clone the Repository">
    Clone the source code to your local machine:

    ```bash
    cd ~/workspace
    git clone <your-repository-url>
    cd source
    ```

    <Note>
      Replace `<your-repository-url>` with your actual Git repository URL.
    </Note>
  </Step>

  <Step title="Install Dependencies">
    Install all required npm packages:

    ```bash
    npm install
    ```

    This will install:
    - React 19 and React DOM
    - Azure MSAL libraries for authentication
    - React Router for navigation
    - Vite for development and building
    - TailwindCSS for styling
    - ESLint for code quality
  </Step>

  <Step title="Configure Azure AD (Optional)">
    If you want to use Azure AD authentication instead of mock authentication, update the `src/authConfig.js` file with your Azure AD credentials:

    ```javascript src/authConfig.js
    export const msalConfig = {
      auth: {
        clientId: "YOUR_CLIENT_ID",
        authority: "https://login.microsoftonline.com/YOUR_TENANT_ID",
        redirectUri: "/",
      },
      cache: {
        cacheLocation: "sessionStorage",
        storeAuthStateInCookie: false,
      }
    };

    export const loginRequest = { 
      scopes: ["User.Read"],
    };
    ```

    <Warning>
      Never commit your actual Azure AD credentials to version control. Use environment variables in production.
    </Warning>
  </Step>

  <Step title="Start the Development Server">
    Launch the Vite development server:

    ```bash
    npm run dev
    ```

    You should see output similar to:

    ```
    VITE v7.1.7  ready in 234 ms

    ➜  Local:   http://localhost:5173/
    ➜  Network: use --host to expose
    ```

    <Tip>
      The development server supports Hot Module Replacement (HMR), so your changes will appear instantly without a full page reload.
    </Tip>
  </Step>

  <Step title="Access the Application">
    Open your browser and navigate to:

    ```
    http://localhost:5173
    ```

    You should see the login page for the Portal Self-Service application.
  </Step>
</Steps>

## Testing with Mock Authentication

The application includes a mock authentication system for development and testing. Here's how to use it:

<Steps>
  <Step title="Start Your Backend Server">
    Ensure your backend API is running on `http://localhost:3000`. The mock authentication expects the following endpoint:

    ```
    GET http://localhost:3000/api/empleados/profile
    ```

    <Info>
      The frontend sends a Bearer token `MOCK_TOKEN_ADMIN` in the Authorization header. Ensure your backend is configured to accept this mock token during development.
    </Info>
  </Step>

  <Step title="Click the Login Button">
    On the login page, click the "Inicio de Sesión" (Login) button. The application will:

    1. Send a request to your backend with the mock token
    2. Retrieve the user profile from the response
    3. Store authentication state in localStorage
    4. Redirect you to the home page

    Here's the authentication flow from `src/pages/LoginPage.jsx`:

    ```javascript src/pages/LoginPage.jsx
    const handleLogin = async () => {
        const mockToken = "MOCK_TOKEN_ADMIN";
        
        try {
            const response = await fetch('http://localhost:3000/api/empleados/profile', {
                method: 'GET',
                headers: {
                    'Authorization': `Bearer ${mockToken}`
                },
                credentials: 'include'
            });

            if (response.ok) {
                const data = await response.json();
                localStorage.setItem('isAuthenticated', 'true');
                localStorage.setItem('userName', data.nombre);
                navigate('/home');
            }
        } catch (error) {
            console.error('Error de red:', error);
        }
    };
    ```
  </Step>

  <Step title="Explore the Home Page">
    After successful authentication, you'll be redirected to `/home` where you can see:

    - A welcome message with your username
    - Confirmation that backend connection was successful
    - A logout button to clear your session

    The home page is protected by the `ProtectedRoute` component:

    ```jsx src/App.jsx
    const ProtectedRoute = ({ children }) => {
        const isAuthenticated = localStorage.getItem('isAuthenticated') === 'true';
      
        if (!isAuthenticated) {
            return <Navigate to="/" replace />;
        }
        
        return children;
    };
    ```
  </Step>
</Steps>

## Testing with Azure AD Authentication

For production-like authentication using Azure AD:

<Steps>
  <Step title="Configure Azure AD Application">
    In the Azure Portal:

    1. Register a new application in Azure Active Directory
    2. Configure the redirect URI to match your local development URL
    3. Enable implicit flow for ID tokens and access tokens
    4. Note your Client ID and Tenant ID
  </Step>

  <Step title="Update authConfig.js">
    Add your Azure AD credentials:

    ```javascript src/authConfig.js
    export const msalConfig = {
      auth: {
        clientId: "d317992c-f1f6-4ecf-8bb5-bc4ad085b979",
        authority: "https://login.microsoftonline.com/d7d9814e-2d7a-4cf4-b34d-b3ad99396e3f",
        redirectUri: window.location.origin,
      },
    };
    ```
  </Step>

  <Step title="Use LoginMicrosoft Component">
    The `src/componentes/LoginMicrosoft.jsx` component handles Azure AD authentication:

    ```javascript src/componentes/LoginMicrosoft.jsx
    const handleLogin = async () => {
      try {
        const response = await instance.loginPopup({
          scopes: ["user.read", "email", "openid", "profile"],
          prompt: "select_account",
        });

        const account = response.account;
        const email = account.username;
        const nombreCompleto = account.name;

        // Store user information
        localStorage.setItem("usuario_email", email);
        localStorage.setItem("usuario_nombre", nombreCompleto);

        navigate("/selfservice");
      } catch (error) {
        console.error("Error en login:", error);
      }
    };
    ```
  </Step>

  <Step title="Test the Login Flow">
    1. Click the "Log In" button
    2. A popup window will appear asking you to select your Microsoft account
    3. Grant the requested permissions
    4. You'll be redirected to the self-service portal
  </Step>
</Steps>

<Note>
  The Azure AD integration also calls backend endpoints to fetch employee data by email and update SSO login records.
</Note>

## Troubleshooting

### Backend Connection Issues

If you see an error about backend connection:

<Warning>
  **Error**: "Error de red. Asegúrate que el backend esté en http://localhost:3000"
</Warning>

**Solution**: Ensure your backend server is running and accessible at the expected URL. Check that:
- The backend is started and listening on the correct port
- No firewall is blocking local connections
- The backend accepts the mock token in development mode

### Azure AD Authentication Errors

If Azure AD login fails:

1. Verify your Client ID and Tenant ID in `authConfig.js`
2. Check that the redirect URI matches your application settings in Azure Portal
3. Ensure your Azure AD app registration has the correct permissions granted
4. Check browser console for detailed error messages from MSAL

### Port Already in Use

If port 5173 is already in use:

```bash
# Vite will automatically try the next available port
# Or specify a custom port:
npm run dev -- --port 3001
```
