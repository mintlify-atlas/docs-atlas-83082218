This guide walks you through setting up the Portal Self-Service application for local development.

## Prerequisites

Before you begin, ensure you have the following installed:

- **Node.js** (version 18 or higher)
- **npm** (comes with Node.js) or **yarn**
- A code editor (VS Code recommended)
- Git for version control

## Initial Setup

<Steps>
  <Step title="Clone the repository">
    Clone the project repository to your local machine:
    
    ```bash
    git clone <repository-url>
    cd portal-selfservice-frontend
    ```
  </Step>

  <Step title="Install dependencies">
    Install all required npm packages:
    
    ```bash
    npm install
    ```
    
    This will install all dependencies listed in `package.json`, including:
    - React 19.1.1
    - Azure MSAL for authentication
    - React Router for navigation
    - Vite for build tooling
    - Tailwind CSS for styling
  </Step>

  <Step title="Configure Azure AD authentication">
    Update the Azure AD configuration in `src/authConfig.js`:
    
    ```javascript src/authConfig.js
    export const msalConfig = {
      auth: {
        clientId: "YOUR_CLIENT_ID", // Replace with your Azure AD Client ID
        authority: "https://login.microsoftonline.com/YOUR_TENANT_ID", // Replace with your Tenant ID
        redirectUri: "/", // Must match registered redirect URI
      },
      cache: {
        cacheLocation: "sessionStorage", // Recommended for SPA
        storeAuthStateInCookie: false,
      }
    };
    
    export const loginRequest = { 
      scopes: ["User.Read"],
    };
    ```
    
    <Note>
      You'll need to obtain the Client ID and Tenant ID from your Azure AD app registration. See the [Azure AD Configuration](/configuration/azure-ad) guide for details.
    </Note>
  </Step>

  <Step title="Start the development server">
    Launch the Vite development server:
    
    ```bash
    npm run dev
    ```
    
    The application will be available at `http://localhost:5173` (default Vite port).
  </Step>
</Steps>

## Development Scripts

The following npm scripts are available in `package.json`:

<CodeGroup>
```bash Development Server
npm run dev
```

```bash Production Build
npm run build
```

```bash Lint Code
npm run lint
```

```bash Preview Production Build
npm run preview
```
</CodeGroup>

### Script Details

| Script | Command | Description |
|--------|---------|-------------|
| `dev` | `vite` | Starts Vite dev server with hot module replacement (HMR) |
| `build` | `vite build` | Creates optimized production build in `dist/` folder |
| `lint` | `eslint .` | Runs ESLint to check code quality |
| `preview` | `vite preview` | Locally preview the production build |

## Development Features

### Hot Module Replacement (HMR)

Vite provides fast HMR out of the box. Changes to your React components will be reflected instantly in the browser without losing component state.

<Info>
  The project uses `@vitejs/plugin-react-swc` which leverages SWC for extremely fast refresh times.
</Info>

### Code Quality Tools

The project includes ESLint configuration for maintaining code quality:

```javascript eslint.config.js
export default defineConfig([
  globalIgnores(['dist']),
  {
    files: ['**/*.{js,jsx}'],
    extends: [
      js.configs.recommended,
      reactHooks.configs['recommended-latest'],
      reactRefresh.configs.vite,
    ],
    languageOptions: {
      ecmaVersion: 2020,
      globals: globals.browser,
    },
    rules: {
      'no-unused-vars': ['error', { varsIgnorePattern: '^[A-Z_]' }],
    },
  },
])
```

### Tailwind CSS Development

Tailwind CSS is configured to scan all source files for class names:

```javascript tailwind.config.js
export default {
  content: [
    "./index.html",
    "./src/**/*.{js,ts,jsx,tsx}",
  ],
  theme: {
    extend: {},
  },
  plugins: [],
}
```
