The Portal Self-Service uses React Router DOM v7 for client-side routing. This guide covers the routing structure, protected routes, and navigation configuration.

## Overview

The application implements a simple two-page structure with authentication protection:

- **Login Page** (`/`) - Public route for authentication
- **Home Page** (`/home`) - Protected route requiring authentication

Routing is configured in `src/App.jsx` using React Router DOM's declarative API.

## Core Routing Configuration

The main routing structure is defined in `src/App.jsx`:

<CodeGroup>
```jsx src/App.jsx
import React from 'react';
import { BrowserRouter, Routes, Route, Navigate } from 'react-router-dom';
import LoginPage from './pages/LoginPage.jsx';
import HomePage from './pages/HomePage.jsx';

const App = () => {
    return (
        <BrowserRouter>
            <Routes>
                {/* Public Route - Login */}
                <Route path="/" element={<LoginPage />} />
                
                {/* Protected Route - Home */}
                <Route 
                    path="/home" 
                    element={
                        <ProtectedRoute>
                            <HomePage />
                        </ProtectedRoute>
                    } 
                />
            </Routes>
        </BrowserRouter>
    );
};

export default App;
```
</CodeGroup>

## Protected Routes

The application implements a custom `ProtectedRoute` component to restrict access to authenticated users:

<CodeGroup>
```jsx src/App.jsx
// Protected Route Component
const ProtectedRoute = ({ children }) => {
    const isAuthenticated = localStorage.getItem('isAuthenticated') === 'true';
  
    if (!isAuthenticated) {
        return <Navigate to="/" replace />;
    }
    
    // If authenticated, render the protected component
    return children;
};
```
</CodeGroup>

### How It Works

<Steps>
  <Step title="Check Authentication">
    The component checks `localStorage` for an `isAuthenticated` flag:
    
    ```javascript
    const isAuthenticated = localStorage.getItem('isAuthenticated') === 'true';
    ```
  </Step>

  <Step title="Redirect if Not Authenticated">
    If the user is not authenticated, redirect to the login page:
    
    ```jsx
    if (!isAuthenticated) {
        return <Navigate to="/" replace />;
    }
    ```
    
    The `replace` prop replaces the current history entry instead of adding a new one.
  </Step>

  <Step title="Render Protected Content">
    If authenticated, render the children (protected page):
    
    ```jsx
    return children;
    ```
  </Step>
</Steps>

## Route Guards Best Practices

<Tip>
**Centralize Authentication Logic**

Keep authentication checks in a single `ProtectedRoute` component for consistency across all protected pages.
</Tip>

<Tip>
**Store Minimal Data**

Only store essential authentication data in localStorage. For sensitive operations, always verify with the backend.
</Tip>

<Tip>
**Handle Redirects Properly**

Use the `replace` prop on `<Navigate>` to prevent users from using the back button to return to protected pages after logout.
</Tip>

<Warning>
**Security Note**: The current authentication implementation uses localStorage for demo purposes. In production, consider using more secure methods like HTTP-only cookies or implementing proper session management with your backend.
</Warning>

## Troubleshooting

### Redirect Loop

**Problem**: Page keeps redirecting in a loop

**Solution**: Check that your authentication state is being set correctly and that the `ProtectedRoute` logic doesn't create circular redirects.

```jsx
// Bad - can cause loops
if (!isAuthenticated) {
    navigate('/'); // Don't use navigate in render
}

// Good - use Navigate component
if (!isAuthenticated) {
    return <Navigate to="/" replace />;
}
```

### Route Not Matching

**Problem**: Route doesn't match even with correct URL

**Solution**: 
1. Check route path spelling
2. Ensure routes are inside `<Routes>` component
3. Check order of routes (specific routes before catch-all)
4. Verify `BrowserRouter` is wrapping your routes

### Protected Route Not Working

**Problem**: Users can access protected pages without authentication

**Solution**: Verify that:
1. `ProtectedRoute` is wrapped around the route element
2. Authentication state is being checked correctly
3. localStorage keys match between login and protection logic
