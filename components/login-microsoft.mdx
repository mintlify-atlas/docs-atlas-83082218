## Overview

The `LoginMicrosoft` component provides enterprise-grade authentication using Microsoft Azure Active Directory (Azure AD). It handles the complete authentication flow including popup login, user account management, session storage, and backend synchronization for employee records.

<Info>
This is the production authentication component for the Portal Self-Service application, replacing simulated authentication with real Azure AD integration.
</Info>

## Full Source Code

```jsx
import { useMsal } from "@azure/msal-react";
import { useEffect } from "react";
import { useNavigate } from "react-router-dom";

function LoginMicrosoft() {
  const { instance, accounts } = useMsal();
  const navigate = useNavigate();

  useEffect(() => {
    if (accounts.length > 0) {
      navigate("/selfservice");
    }
  }, [accounts, navigate]);

  const handleLogin = async () => {
    try {
      const response = await instance.loginPopup({
        scopes: ["user.read", "email", "openid", "profile"],
        prompt: "select_account",
      });

      const account = response.account;
      const email = account.username;
      const nombreCompleto = account.name;

      const empleadoResponse = await fetch(
        `http://localhost:5000/api/empleados/email/${email}`
      );
      const empleadoData = await empleadoResponse.json();

      if (empleadoData?.id_empleado) {
        await fetch("http://localhost:5000/api/sso/actualizar-login", {
          method: "POST",
          headers: { "Content-Type": "application/json" },
          body: JSON.stringify({ id_empleado: empleadoData.id_empleado }),
        });
      }

      localStorage.setItem("usuario_email", email);
      localStorage.setItem("usuario_nombre", nombreCompleto);

      navigate("/selfservice");
    } catch (error) {
      console.error("Error en login:", error);
    }
  };

  const handleLogout = () => {
    instance.setActiveAccount(null);
    localStorage.removeItem("usuario_email");
    localStorage.removeItem("usuario_nombre");
    navigate("/");
  };

  return (
    <div className="flex flex-col items-center justify-center h-screen bg-gray-50">
      <div className="w-full max-w-md p-5 text-center bg-white shadow-md rounded-xl">
        <h1 className="mb-6 text-2xl font-bold text-gray-700">
          Sistema de Gestión de Empleados
        </h1>
        {accounts.length === 0 ? (
          <>
            <p className="mb-4 text-gray-600">
              Inicia sesión con tu correo de ELEOS
            </p>
            <button
              onClick={handleLogin}
              className="px-6 py-3 text-white transition bg-blue-600 rounded-lg hover:bg-blue-700"
            >
              Log In
            </button>
          </>
        ) : (
          <>
            <p className="mb-4 text-gray-600">
              Sesión activa como:
              <br />
              <strong>{accounts[0].name}</strong>
            </p>
            <button
              onClick={handleLogout}
              className="px-6 py-3 text-white transition bg-red-600 rounded-lg hover:bg-red-700"
            >
              Log Out
            </button>
          </>
        )}
      </div>
    </div>
  );
}

export default LoginMicrosoft;
```

## Component Features

### Azure AD Authentication

Integrates with Microsoft Authentication Library (MSAL) for secure OAuth 2.0 / OpenID Connect authentication.

### Popup-based Login

Uses popup window for authentication to avoid full page redirects and improve user experience.

### Automatic Redirect

Automatically redirects authenticated users to the self-service portal.

### Backend Synchronization

Syncs login events with backend to track employee access and update last login timestamps.

### Session Management

Stores user information in localStorage for application-wide access.

### Dual-state UI

Dynamically displays login or logout interface based on authentication state.

## Functions

### handleLogin

```javascript
const handleLogin = async () => {
  try {
    const response = await instance.loginPopup({
      scopes: ["user.read", "email", "openid", "profile"],
      prompt: "select_account",
    });
    // ... rest of logic
  } catch (error) {
    console.error("Error en login:", error);
  }
};
```

**Purpose:** Manages the complete login flow from authentication to backend sync

**Parameters:**
- **scopes**: Array of Microsoft Graph API permissions requested
- **prompt**: `"select_account"` ensures user can choose which account to use

**Error Handling:** Catches and logs authentication errors without crashing the application

<Accordion title="Common Login Errors">
- `user_cancelled`: User closed the popup or cancelled login
- `consent_required`: User needs to consent to requested scopes
- `interaction_required`: Additional interaction needed (e.g., MFA)
- Network errors: Backend unavailable
</Accordion>

### handleLogout

```javascript
const handleLogout = () => {
  instance.setActiveAccount(null);
  localStorage.removeItem("usuario_email");
  localStorage.removeItem("usuario_nombre");
  navigate("/");
};
```

**Purpose:** Terminates user session and clears all authentication data

**Actions:**
1. Clear MSAL active account
2. Remove email from localStorage
3. Remove name from localStorage
4. Navigate back to login page

<Note>
This performs a local logout only. For complete sign-out including Azure AD session, use `instance.logoutPopup()` or `instance.logoutRedirect()`.
</Note>

## MSAL Configuration Requirements

The component requires MSAL to be configured in a parent component:

```javascript
import { PublicClientApplication } from '@azure/msal-browser';
import { MsalProvider } from '@azure/msal-react';

const msalConfig = {
  auth: {
    clientId: "your-azure-ad-client-id",
    authority: "https://login.microsoftonline.com/your-tenant-id",
    redirectUri: "http://localhost:3000",
  },
  cache: {
    cacheLocation: "sessionStorage",
    storeAuthStateInCookie: false,
  }
};

const pca = new PublicClientApplication(msalConfig);

function App() {
  return (
    <MsalProvider instance={pca}>
      <LoginMicrosoft />
    </MsalProvider>
  );
}
```

<Warning>
**Azure AD App Registration Required:**

Before using this component, you must:
1. Register an application in Azure Portal
2. Configure redirect URIs
3. Set API permissions (User.Read, email, openid, profile)
4. Note the Client ID and Tenant ID
</Warning>

## Session Storage

The component stores user information in localStorage:

```javascript
localStorage.setItem("usuario_email", email);
localStorage.setItem("usuario_nombre", nombreCompleto);
```

**Stored Data:**
- **usuario_email**: User's email from Azure AD (e.g., "user@eleos.com")
- **usuario_nombre**: User's display name (e.g., "Juan Pérez")

**Usage in Other Components:**
```javascript
const userEmail = localStorage.getItem('usuario_email');
const userName = localStorage.getItem('usuario_nombre');
```

<Warning>
**Security Consideration:**

localStorage is vulnerable to XSS attacks. Consider:
- Storing only non-sensitive data
- Using httpOnly cookies for tokens
- Implementing proper Content Security Policy (CSP)
</Warning>

const fetchWithRetry = async (url, options, retries = 3) => {
  for (let i = 0; i < retries; i++) {
    try {
      const response = await fetch(url, options);
      if (response.ok) return response;
    } catch (error) {
      if (i === retries - 1) throw error;
      await new Promise(resolve => setTimeout(resolve, 1000 * (i + 1)));

## Usage Example

### Complete Application Setup

```jsx
import React from 'react';
import ReactDOM from 'react-dom/client';
import { BrowserRouter, Routes, Route } from 'react-router-dom';
import { PublicClientApplication } from '@azure/msal-browser';
import { MsalProvider } from '@azure/msal-react';

import LoginMicrosoft from './componentes/LoginMicrosoft';
import SelfService from './componentes/SelfService';

const msalConfig = {
  auth: {
    clientId: process.env.REACT_APP_AZURE_CLIENT_ID,
    authority: `https://login.microsoftonline.com/${process.env.REACT_APP_AZURE_TENANT_ID}`,
    redirectUri: window.location.origin,
  },
  cache: {
    cacheLocation: "sessionStorage",
    storeAuthStateInCookie: false,
  }
};

const pca = new PublicClientApplication(msalConfig);

function App() {
  return (
    <MsalProvider instance={pca}>
      <BrowserRouter>
        <Routes>
          <Route path="/" element={<LoginMicrosoft />} />
          <Route path="/selfservice" element={<SelfService />} />
        </Routes>
      </BrowserRouter>
    </MsalProvider>
  );
}

const root = ReactDOM.createRoot(document.getElementById('root'));
root.render(<App />);
```

## Security Best Practices

<Tabs>
  <Tab title="Token Handling">
    Consider acquiring and passing MSAL access tokens to backend:
    
    ```javascript
    const request = {
      scopes: ["user.read"],
      account: accounts[0]
    };
    
    const tokenResponse = await instance.acquireTokenSilent(request);
    
    fetch(url, {
      headers: {
        'Authorization': `Bearer ${tokenResponse.accessToken}`
      }
    });
    ```
  </Tab>
  <Tab title="Validate on Backend">
    Backend must validate:
    - Token signature
    - Token expiration
    - User permissions
    - Email matches database record
  </Tab>
  <Tab title="Secure Storage">
    - Use httpOnly cookies for sensitive tokens
    - Keep localStorage for display data only
    - Implement proper CORS policies
    - Use HTTPS in production
  </Tab>
  <Tab title="Session Management">
    - Implement session timeout
    - Refresh tokens before expiration
    - Handle token renewal gracefully
    - Clear all data on logout
  </Tab>
</Tabs>
