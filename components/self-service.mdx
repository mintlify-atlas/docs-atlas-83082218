## Overview

The `SelfService` component displays authenticated employee information by fetching data from the backend using the user's Azure AD email. It integrates with Microsoft Authentication Library (MSAL) to retrieve the logged-in user's identity and displays their complete profile information.

<Info>
This component is designed for production use with Azure AD authentication. It automatically retrieves user data based on the authenticated Microsoft account.
</Info>

## Full Source Code

```jsx
import { useMsal } from "@azure/msal-react";
import { useEffect, useState } from "react";

function SelfService() {
  const { accounts } = useMsal();
  const [userData, setUserData] = useState(null);

  useEffect(() => {
    if (accounts.length > 0) {
      const userEmail = accounts[0].username;

      fetch(`http://localhost:5000/api/empleados/email/${userEmail}`)
        .then((res) => {
          if (!res.ok) {
            throw new Error(`Error HTTP: ${res.status}`);
          }
          return res.json();
        })
        .then((data) => setUserData(data))
        .catch((err) => {
          console.error("Error al obtener datos del usuario:", err);
          setUserData({ error: "No se pudo cargar la información." });
        });
    }
  }, [accounts]);

  if (!userData) {
    return (
      <div className="mt-10 text-center text-gray-600">
        Cargando información...
      </div>
    );
  }

  if (userData.error) {
    return (
      <div className="mt-10 text-center text-red-600">
        {userData.error}
      </div>
    );
  }

  return (
    <div className="max-w-lg p-6 mx-auto mt-10 bg-white shadow-md rounded-xl">
      <h2 className="mb-4 text-2xl font-bold text-center">Mi Perfil</h2>
      <div className="space-y-2">
        <p><strong>Nombre:</strong> {userData.nombre}</p>
        <p><strong>Correo:</strong> {userData.correo}</p>
        <p><strong>DNI:</strong> {userData.DNI}</p>
        <p><strong>Teléfono:</strong> {userData.telefono}</p>
        <p><strong>Clínica:</strong> {userData.clinica}</p>
        <p><strong>Estado:</strong> {userData.estado}</p>
        <p>
          <strong>Fecha de ingreso:</strong>{" "}
          {new Date(userData.fecha_ingreso).toLocaleDateString()}
        </p>
      </div>
    </div>
  );
}

export default SelfService;
```

## Component Lifecycle

<Accordion title="Mount Phase">
1. Component renders with `userData = null`
2. Displays "Cargando información..."
3. `useEffect` hook executes
4. Checks if `accounts.length > 0`
5. If accounts exist, fetches employee data
</Accordion>

<Accordion title="Data Fetching Phase">
1. Extract user email from `accounts[0].username`
2. Initiate fetch request to backend
3. Wait for response
4. Parse JSON or catch errors
5. Update `userData` state
6. Component re-renders with new data
</Accordion>

<Accordion title="Update Phase">
If `accounts` array changes (new login, logout, etc.):
1. `useEffect` runs again
2. Fetches data for new account
3. Updates display
</Accordion>

```jsx
import { MsalProvider } from '@azure/msal-react';
import { PublicClientApplication } from '@azure/msal-browser';

const msalConfig = {
```

## Security Considerations

<Warning>
**Important Security Notes:**

1. **Backend Validation**: The backend must verify the user's identity, not just accept any email parameter
2. **CORS Configuration**: Ensure proper CORS settings for `http://localhost:5000`
3. **Data Exposure**: Only expose employee data that the user should see
4. **Authentication Token**: Consider passing MSAL token to backend for verification
</Warning>

### Recommended Improvement

```javascript
const { instance, accounts } = useMsal();

// Acquire token silently
const request = {
  scopes: ["user.read"],
  account: accounts[0]
};

const response = await instance.acquireTokenSilent(request);
const accessToken = response.accessToken;

// Include token in fetch
fetch(`http://localhost:5000/api/empleados/email/${userEmail}`, {
  headers: {
    'Authorization': `Bearer ${accessToken}`
  }
})
```

## Performance Considerations

<Accordion title="Preventing Unnecessary Fetches">
The `useEffect` dependency array includes `accounts`, which means the fetch runs whenever accounts change. Consider adding a check:

```javascript
const [lastFetchedEmail, setLastFetchedEmail] = useState(null);

useEffect(() => {
  if (accounts.length > 0) {
    const userEmail = accounts[0].username;
    
    // Only fetch if email changed
    if (userEmail !== lastFetchedEmail) {
      // ... fetch logic
      setLastFetchedEmail(userEmail);
    }
  }
}, [accounts, lastFetchedEmail]);
```
</Accordion>

<Accordion title="Adding Cache">
```javascript
const [cache, setCache] = useState({});

if (cache[userEmail]) {
  setUserData(cache[userEmail]);
} else {
  fetch(...)
    .then(data => {
      setUserData(data);
      setCache(prev => ({ ...prev, [userEmail]: data }));
    });
}
```
</Accordion>
