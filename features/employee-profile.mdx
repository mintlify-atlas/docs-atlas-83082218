The employee profile feature allows authenticated users to view their personal information retrieved from the backend database. This feature is implemented in the `SelfService` component and integrates with Azure AD authentication.

## Overview

The employee profile displays comprehensive information about the logged-in employee, including:

- Full name
- Email address
- DNI (National ID)
- Phone number
- Assigned clinic
- Employment status
- Date of hire

<Info>
  The profile data is fetched from the backend API using the authenticated user's email address.
</Info>

## Implementation

The employee profile is implemented in the `SelfService` component:

```jsx SelfService.jsx
import { useMsal } from "@azure/msal-react";
import { useEffect, useState } from "react";

function SelfService() {
  const { accounts } = useMsal();
  const [userData, setUserData] = useState(null);

  useEffect(() => {
    if (accounts.length > 0) {
      const userEmail = accounts[0].username;

      fetch(`http://localhost:5000/api/empleados/email/${userEmail}`)
        .then((res) => {
          if (!res.ok) {
            throw new Error(`Error HTTP: ${res.status}`);
          }
          return res.json();
        })
        .then((data) => setUserData(data))
        .catch((err) => {
          console.error("Error al obtener datos del usuario:", err);
          setUserData({ error: "No se pudo cargar la información." });
        });
    }
  }, [accounts]);

  if (!userData) {
    return (
      <div className="mt-10 text-center text-gray-600">
        Cargando información...
      </div>
    );
  }

  if (userData.error) {
    return (
      <div className="mt-10 text-center text-red-600">
        {userData.error}
      </div>
    );
  }

  return (
    <div className="max-w-lg p-6 mx-auto mt-10 bg-white shadow-md rounded-xl">
      <h2 className="mb-4 text-2xl font-bold text-center">Mi Perfil</h2>
      <div className="space-y-2">
        <p><strong>Nombre:</strong> {userData.nombre}</p>
        <p><strong>Correo:</strong> {userData.correo}</p>
        <p><strong>DNI:</strong> {userData.DNI}</p>
        <p><strong>Teléfono:</strong> {userData.telefono}</p>
        <p><strong>Clínica:</strong> {userData.clinica}</p>
        <p><strong>Estado:</strong> {userData.estado}</p>
        <p>
          <strong>Fecha de ingreso:</strong>{" "}
          {new Date(userData.fecha_ingreso).toLocaleDateString()}
        </p>
      </div>
    </div>
  );
}

export default SelfService;
```

### Loading State

While data is being fetched, the component displays a loading message:

```jsx
if (!userData) {
  return (
    <div className="mt-10 text-center text-gray-600">
      Cargando información...
    </div>
  );
}
```

### Error State

If the API call fails, an error message is displayed:

```jsx
if (userData.error) {
  return (
    <div className="mt-10 text-center text-red-600">
      {userData.error}
    </div>
  );
}
```

### Success State

When data is successfully loaded, the profile is displayed in a styled card with proper formatting.

## API Integration

### Endpoint

The component fetches employee data from the following endpoint:

```
GET /api/empleados/email/{email}
```

### Request Example

```javascript
fetch(`http://localhost:5000/api/empleados/email/john.doe@example.com`)
  .then((res) => res.json())
  .then((data) => console.log(data));
```

### Expected Response

The API should return a JSON object with the following structure:

```json
{
  "id_empleado": 123,
  "nombre": "John Doe",
  "correo": "john.doe@example.com",
  "DNI": "12345678",
  "telefono": "+1234567890",
  "clinica": "Main Clinic",
  "estado": "Active",
  "fecha_ingreso": "2024-01-15T00:00:00.000Z"
}
```

<Note>
  The `fecha_ingreso` field is automatically formatted using JavaScript's `toLocaleDateString()` method to display in the user's locale format.
</Note>

Each field is displayed in a consistent format:

```jsx
<p><strong>Field Label:</strong> {fieldValue}</p>
```

Special formatting is applied to:

- **Dates**: Converted to local date string format
- **All text**: Displayed as-is from the API response

## Customization Options

### Adding New Fields

To display additional employee information:

1. Ensure the backend API returns the new field
2. Add a new line in the profile display:

